## 1. 背景判断

近期参考 AI Dungeon 的内容流后，一个重要结论是：

**产品级不一定来自图片画风完全统一，而是来自图片被同一套产品系统和后期规范收住。**

AI Dungeon 的图片本身可以很杂：动漫、厚涂幻想、写实场景、校园、童话、meme 感内容都能并列出现。它仍然成立，原因不是图片都像同一个画师画的，而是图片比例、裁切、明暗、卡片容器、标题和数据呈现形成了统一的产品框架。

InkWild 不必追求所有图片都像同一种 AI 风格，也不必让原作类世界复刻官方影视海报。更合适的方向是建立自己的“故事封面系统”。

## 1.1 House Texture：统一的是媒介，不是风格（v0.2 核心）

更深一层的诊断（对比 AI Dungeon 内容流 vs 本站 discover 实拍）：

AI Dungeon 的图放一起舒服，**真正的原因不是风格统一，而是媒介统一**——不管动漫、厚涂、校园、写实场景，它们底层都是「画」（painterly / 概念美术 / 插画），即使偏写实的题材也是绘画质感，不是照片。它们流派不同，但**媒介相同**。

本站 discover 之所以「脏、杂、廉价」，根因是图片底层是 **photorealistic AI render**：

- 写实渲染是最容易暴露 AI 味的媒介——塑料反光、HDR 过曝、过度锐化、细节噪点堆叠。
- 不同题材的写实照片**没有共同调性**：水墨灰雾 + 写实人脸 + 赛博朋克霓虹放一起，彻底炸开。

所以 InkWild 要统一的不是「风格」，是 **「媒介质感 = 绘画，不是照片」**：

- **地基**：所有图都是绘画 / 概念美术 / 封面插画质感，有笔触、有肌理、手绘感。
- **允许多样**：题材色温与画法（厚涂、水彩、淡彩、版画、工笔、水墨、赛璐珞…）各自保留，由模型按题材气质选。
- **结果**：多样但坐在同一地基上 → 「多样但收得住」，而不是「杂乱无章」。

> 对照参考：login 左图（版画/单色文学插画）恰好证明 InkWild 显贵的方向就是插画质感——它只是手工素材，现在要把这套质感推广到全部生图产物。

这条是 §2 之上的总纲：下面 §2-§7 讲的「文学封面感」「统一后期」「题材转译」都要先坐在插画地基上才成立。

### 1.2 明度：统一媒介，但明暗交给题材（不要统一提亮，也不要统一压黑）

落地时踩过一个反复（cover prompt A/B 实测，2026-06-26）：先是 prompt 隐含压暗 → covers 普遍发暗发脏；于是反向加了「默认明快、画面要亮才从近黑网站跳出来」想提亮。**两个方向都错。**

关键发现：**最初"暗、脏"的病根是「写实媒介 + 压暗」，不是"暗"本身。** 一旦媒介钉死为插画，"暗"就变成高级的「插画的暗」（厚涂/水彩笔触、暗部仍有色彩），一点不脏。这时候：

- **明度根本不用规范去管**：模型自己比我们更懂每个题材该亮该暗（晨景书院亮、悬疑中暗、赛博暗夜暗）。
- **"默认提亮"是个偏见**：它会把本该暗的赛博/恐怖/末日往亮里拽，削弱暗黑气质——正好违反 §3.3「不为明亮改写暗题材气质」。

所以最终规则：**媒介=插画（钉死）＋ 明暗基调=完全交给题材**，prompt 里只留一句纯防呆的网页可读底线（暗部别糊成死黑一团、高光别过曝），不带任何"亮/暗"导向。少即是多。代码见 `cover_brief.py` 的 `_TONE_FREEDOM`。

## 2. 核心方向

**InkWild House Style = 文学封面感 + 叙事剧照 + 统一后期。**

这意味着：

- 允许不同题材拥有不同画风。
- 不要求所有世界像同一部作品、同一位画师、同一套模型预设。
- 不追求原作复刻，而是把原作或题材转译成 InkWild 自己的互动故事封面。
- 统一重点放在比例、构图、裁切安全、色彩后期、质感控制和禁区约束上。

一句话描述：

**一座故事图书馆里的互动封面。**

## 3. 不再追求的方向

### 3.1 不追求绝对画风统一

悬疑可以偏灰雾和低饱和，奇幻可以更明亮，校园可以偏动画，现代都市可以更写实。只要进入同一套后期和裁切系统，就可以共存。

不建议把所有内容强行统一成：

- 纯水墨
- 纯国风
- 纯暗黑奇幻
- 纯影视剧照
- 纯二次元
- 纯 AI 概念图

### 3.2 不追求原作外观复刻

原作类世界不要追求“长得像官方海报/影视截图”。更安全也更有品牌感的处理是题材转译：

| 原始表达 | InkWild 表达 |
|---|---|
| 哈利波特电影风 | 魔法学院悬疑封面 |
| 甄嬛传剧照风 | 清宫权谋故事封面 |
| 庆余年官方海报风 | 架空朝堂谍战封面 |
| 东方快车经典电影风 | 雪夜列车密室谜案封面 |

这样能保留用户对题材的理解，同时减少廉价仿制感和 IP 风险。

### 3.3 不靠压黑解决风格差异

过度黑膜会让不同图片统一，但代价是画面发灰、发闷、没有空气感。InkWild 需要的是“统一后期”，不是“统一盖黑”。

建议避免：

- 全图大面积 40% 以上黑色遮罩作为常规方案。
- 图片整体降亮到细节丢失。
- 所有题材都压成同一种暗黑气氛。

## 4. 应该统一的内容

### 4.1 统一比例

沿用现有产物规格：

| 图片类型 | 比例 | 用途 |
|---|---|---|
| World hero | 21:9 | 大幅沉浸展示 |
| World cover | 3:2 | 世界列表 / 卡片陈列 |
| Script cover | 3:2 | 剧本列表 / 剧本精选 |
| Ending card | 3:2 | 结局展示 |
| Character portrait | 2:3 | 角色头像 / 角色资料 |

Hero 和 cover 不应强行共用同一张图。Hero 需要横向场景和留白，cover 需要小尺寸下仍有清晰主体。

### 4.2 统一构图纪律

每张图都应该有一个明确的“叙事钩子”：

- 一个即将进入的地点
- 一个关键道具
- 一次事件发生前后的瞬间
- 一个背影、侧影或小群像
- 一束能引导视线的光
- 一个能暗示谜题/冲突的空间

避免：

- 只有氛围，没有主体
- 人物太多，缩小后不可读
- 纯风景明信片
- 过满的 AI 奇观堆叠
- 中央主体被 UI 或裁切遮挡
- 画面中文字、logo、伪海报信息抢走 UI 标题职责

### 4.3 统一后期

不同画风可以共存，但进入 InkWild 后应经过类似的后期方向：

- 轻微降饱和，不要糖果色失控
- 保留局部亮点，不要整体压暗
- 边缘轻微暗角，中心或主体保留空气
- 对比度柔化，避免 AI 图常见的硬锐边
- 暖光可以作为品牌气质，但不要把所有图染成金色
- 黑位要有层次，不能全糊成一块

可理解为：

**不是把图片变黑，而是把图片变得像一本同系列故事集里的插图。**

## 5. 各图片类型建议

### 5.1 World hero

目标：建立世界气氛，让用户愿意进入。

建议：

- 21:9 横向大场景。
- 左侧或下方保留安全区，方便前端叠加标题/按钮时可读。
- 有地点感，最好能看出“入口”：门、桥、街口、车站、楼、庭院、山门、船舱、走廊。
- 不要只拍人物大脸；hero 更适合世界空间。
- 不要依赖画面内文字说明，文字应主要交给 UI。

### 5.2 World cover

目标：在卡片列表里快速识别世界。

建议：

- 3:2，主体更大、更明确。
- 小尺寸下也能读懂：一个人物/地点/道具/冲突即可。
- 不要把信息都塞进远景。
- 不要用太细碎的群像。
- 尽量避免和 hero 完全同构，cover 应该更像“书封面”，hero 更像“场景剧照”。

### 5.3 Script cover

目标：表达剧本的任务、冲突或谜题。

建议：

- 比 world cover 更具体。
- 能暗示“这一局要解决什么”：信件、案发现场、失踪线索、宴席、列车包厢、密室、账本、地图、证物。
- 可以更接近悬疑/任务海报，但不要出现伪官方海报排版。
- 不要把剧本标题烙在画面里，除非后续明确设计画面级标题系统。

### 5.4 Ending card

目标：让结局有记忆点。

建议：

- 象征性表达结果，不直白展示血腥或死亡。
- 适合用空场、道具、光线、远景背影表达余韵。
- 结局图不需要比 cover 更刺激，重点是“收束感”。

### 5.5 Character portrait

目标：识别角色气质，不追求真人脸复刻。

建议：

- fictional face，避免真实明星/公众人物可识别面孔。
- 优先半身、侧脸、三分之二角度。
- 角色年龄较小时避免清晰正脸特写，可用侧脸/背影/局部。
- 服装、姿态、道具比脸更重要。

## 6. 题材转译建议

| 题材 | 推荐视觉关键词 | 避免 |
|---|---|---|
| 悬疑 / 推理 | 雾、局部光、线索道具、空房间、门缝、雨夜 | 血腥尸体、廉价惊悚脸 |
| 古风 / 权谋 | 庭院、屏风、烛光、书信、朝堂远景、背影 | 官方剧照复刻、满屏金碧辉煌 |
| 武侠 / 江湖 | 山门、渡口、雨巷、剑匣、客栈、远行背影 | 过度玄幻特效 |
| 奇幻 / 学院 | 建筑入口、图书馆、光源、地图、仪式感 | 游戏宣传图式爆炸特效 |
| 现代 / 都市 | 街口、室内光、地铁、办公室、窗边人物 | 普通 stock photo 感 |
| 校园 / 日常 | 教室、走廊、社团房、黄昏操场、局部人物关系 | 过度模板二次元表情 |
| 科幻 | 设备、舱门、城市远景、冷暖光对照 | 纯蓝色 HUD 堆满 |

## 7. Prompt 方向词

可作为未来 prompt builder 或手工回炉时的方向参考：

```text
interactive story cover, literary book cover sensibility, narrative still,
clear story hook, cinematic but restrained, readable at small card size,
one strong subject, atmospheric depth, soft contrast, subtle vignette,
muted but not dark, no logo, no watermark, no official poster layout,
no celebrity likeness, no fake typography, no excessive text in image
```

中文方向：

```text
互动故事封面，文学封面感，叙事剧照感，有明确故事钩子，
小尺寸卡片中仍可识别，主体清楚，气氛克制，柔和对比，
轻微暗角，保留局部光感，低饱和但不压黑，
不要 logo，不要水印，不要官方海报排版，不要明星脸，不要伪标题字
```

## 8. 评审检查表

一张图是否适合作为 InkWild 图片资产，可以快速问：

- 缩小成卡片后还能一眼看懂吗？
- 是否有一个明确的叙事钩子？
- 是否像“互动故事封面”，而不是普通 AI 氛围图？
- 是否保留了题材差异，而不是被统一压黑？
- 是否避免了原作官方物料复刻？
- 是否没有真实品牌、logo、水印、伪发行信息？
- 是否没有过度依赖画面内文字？
- 和其他图放在一起时，是“多样但被收住”，还是“杂乱无章”？

## 9. 与现有规范的关系

本文不替代 [`cover-art-spec.md`](cover-art-spec.md)：

- `cover-art-spec.md` 负责硬规格：产物类型、比例、格式、法律/安全红线、命名和回炉流程。
- 本文负责审美方向：允许多风格、统一后期、题材转译、构图原则。

如果两者冲突，短期以 `cover-art-spec.md` 和当前代码为准；本文作为下一轮图片系统探索和 prompt 改造的方向依据。

## 10. 落地路线（v0.2 起）

按杠杆从大到小、成本从低到高排：

| 阶段 | 内容 | 状态 |
|---|---|---|
| ① prompt 质感地基 | `cover_brief.py` 改插画地基（§1.1）+ 明暗交给题材（§1.2），治「廉价写实味」+「全暗」根因 | ✅ 已落 + 3 轮 cover A/B 真跑验证（2026-06-26，未 commit） |
| ② 前端统一呈现层 | 图片呈现收敛成组件级固定规则：统一裁切、统一底部渐变/暗角蒙层、统一 hover、统一 fallback；**出图失败必须兜底图，不显破图占位**（discover 现有「图片错误」占位很伤陈列）。治「前端在各页手调、不可控」 | 🔵 待做 |
| ③ 统一后期 LUT | §4.3 的「统一后期」目前是空话——无任何代码执行。出图后过同一道调色/暗角/降噪，把多样图「显影」成同一系列。这是上到 AI Dungeon 级一致性的关键 | 🔵 待做（第二阶段） |
| ④ 文字烙印兜底 | `_LOGO_NEGATIVE` 已有但仍漏网（如「大唐狄云」烙进画面）；插画地基本身会减少，再加出图后 OCR 检测回炉 | 🔵 待做 |

验证方法：阶段①落地后，挑 2-3 个现存世界（建议覆盖悬疑/古风/赛博朋克各一）走草稿页「AI 重抽」，对比新旧封面是否从「写实照片」转为「插画」，再决定是否批量回炉存量世界。
