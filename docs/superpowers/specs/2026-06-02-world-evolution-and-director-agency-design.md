# 世界级演化与导演能动性 —— 架构分析与解法方向

状态：分析完成（2026-06-02），方向待拍板（尚未进入实现计划）
来源：① 修复（"玩家强推→拿到挣来的 authored 结局"）落地后的真跑验证 + 自由模式两局探测（IP 契合 / 反事实骨气）。本文把验证中暴露的两个深层问题、共同根因、解法架构与分期讲清楚。
读法：现象 → 验证（带代码取证）→ 为什么难 → 解法架构 → 分期与优先级。

---

## 0. 速览

真跑验证里冒出两个表层看不同、根上同源的问题：

1. **剧本模式 · 导演太被动**：玩家不会玩 / 跑偏时，导演不主动把他往案件 crux 上拽 → 只能拖到 stall 地板发安慰奖。
2. **全模式 · 世界不吸收结构变更**：玩家塞反常识 / 越界断言（"我已是皇后"），世界既不照单全收、也不正面回应（躲进环境氛围）。**更深一层**：哪怕是**合法挣来**的结构变化（甄嬛真的上位当皇后），现状也**吸收不了**。

一句话根因：**为了防 railroad，导演被过校正成"只描述、不推动、不改写"的旁观者；同时引擎的世界"脊柱"是每回合从种子重建的定格照，没有可持久演化的结构层。** 于是世界只在"软层"运转，"硬层"冻结——拒绝假断言和吸收不了真演化，是同一枚硬币。

---

## 1. 验证：引擎世界态 = 一层"活的" + 一层"冻的"

| 层 | 内容 | 是否随游戏持久演化 + 回灌 prompt | 代码 |
|---|---|---|---|
| **动态软层** | 地点/时间、线索、物品、**NPC 关系(trust/mood)**、NPC 意图/位置、已触发事件、world_conflicts、案件板、narrative_arc | ✅ 持久；每回合作为 user 消息喂导演/NPC，NPC 真会记仇/转变态度 | `engine/state_manager.py:65-115`（GameState 字段）；npc_relations 表 |
| **`world_state` 键值** | 作者预声明的旗标（如 `world_state.huafei_exposed`）| 🟡 **只能沿作者预埋的轴**演化：靠 authored event 的 effects 翻，只有剧本模式有 | `engine/world_simulator.py:206-233`（`apply_intent_effects` 写 world_state/npc_locations）；`engine/condition_tree.py:56`（DSL 只认 `world_state.<key>` 且校验 allowed_keys）|
| **静态脊柱** | `base_setting`、`script_setting`（世界观/真相）、**NPC 人设/身份/位份/谁活着/谁是皇后** | ❌ **每回合从 DB 种子重建，游戏中从不被改写** | `services/game_service.py:_load_world_data`（每回合重载 world_data）；`engine/prompts.py:795+ build_director_system_v2`（脊柱进系统提示，prefix-cache 友好）|

**两条关键事实：**
- **导演没有"改世界结构"的通道**。它能改关系/线索/位置/时间，或在剧本模式**触发作者预写的事件**；但**不能凭空写入一条新结构事实**（"华妃失势""甄嬛晋贵妃""谁死了"）。NPC 人设永远从种子读——剧情里某人"死了"，下一回合导演看到的还是活着的人设。
- **自由模式 = 冻结脊柱 + 软覆盖**：无 script/events → world_state 机制根本不参与 → 结构层完全定格。

**实测佐证（自由模式·反事实局，50 轮）**：5 次硬塞结构断言（我是皇后 / 已废宜修 / 华妃已死 / 我护温实初 / 我怀太子），回应全是环境氛围（"空气先凝住了""抹布落地"），无 NPC 承认也无 NPC 驳斥；而世界态从未改写——非注入回合玩家自己仍称"臣妾"向皇后请安、华妃的人周宁海/剪秋照常在场。**世界靠"继续按种子真实叙事"把假断言悄悄架空（软骨气/惯性维持），代价是真演化也焊不进去。**

## 2. 为什么"表面没事，深层炸"

日常游玩与测试几乎全落在**软层**（关系/情绪/线索/位置）——那层是活的，体验上"世界在运转"。只有玩家去碰**结构层**（身份/位份/生死/权力）才暴露脊柱定格。所以这是一类**低频但根上**的缺陷：覆盖测试碰不到、Demo 演示碰不到，只有较劲的玩家踩中。

同理，"软骨气"（NPC 对挑衅躲进氛围而不反应）也属同一根：导演被设计成"客观描述刺激、NPC 自决"（`prompts.py:818-828`），结果 NPC 对"自称皇后"这种诛心之言"选择沉默"——既不是机制 bug，也不是好体验。

## 3. 真正难的四件事（不是"让它可变"那么简单）

天真解法（让导演往持久库写任意世界事实、prompt 再读）会立刻撞四堵墙：

1. **【仲裁】** 怎么分"甄嬛 30 回合处心积虑挣来的皇后" vs "玩家打字'我是皇后'"？放行 = yes-man（最怕的），全卡 = 回到现在。这是**最难的判断**，且与"判结局是否满足 soft_conditions"是同一种判断。
2. **【自由模式没有 ground truth】** 剧本模式作者能预定义"上位条件"（类比 soft_conditions）；自由模式**没有真相可对**，拿什么判"合法"？这是自由模式的死结。
3. **【级联一致性】** 一个结构变更是全局涟漪：华妃一死 → 从在场名单移除、关系冻结、涉她的事件作废、他人有反应、权力真空重排。写个 flag 不会自己传导。
4. **【IP 漂移】** 产品最强资产是 IP 契合（评测全场最高维 4.2–4.65）。第 5 回合就让甄嬛当皇后，世界便不再是甄嬛传。**"活世界"与"IP 忠诚"天然打架**——而且适度"冻"其实是个**特性**（保叙事连贯 + IP 忠诚），目标不是"什么都能改"。

## 4. 解法架构：一层"被仲裁的结构账本"覆盖在种子上

核心：**脊柱从"纯种子"改为"种子 + 持久结构账本"，每回合重建时叠账本；账本只接受仲裁过的合法变更。**

三个件：

### 4.1 结构账本（持久层）
- 每条结构事实带 `{fact, effective_round, provenance}`，如 `华妃=赐死@r12`、`甄嬛=贵妃@r30`、`皇后=宜修→废@r40`。
- Prompt 构建时把账本叠到种子上 → "华妃年世兰（已于第 12 回合被赐死，现已不在人世）"。**脊柱 = 种子 + 账本**。
- 落点：新增持久字段（GameState 扩展或独立表），`_load_world_data` / `build_director_system_v2` 在拼脊柱时套用 overlay。注意 **prefix-cache**：account 变更不频繁，可把"种子 + 账本"快照化以尽量保留缓存。

### 4.2 世界仲裁闸（心脏）
- 任一结构变更被提议（导演提交，或玩家动作累积触发）→ 仲裁判：**在已积累的局势下，这是挣来的、合理的，还是凭空 fiat？**
- 合法 → 写账本，全世界从此都认；
- fiat / 不合理 → 拒，且**让世界有反应**（NPC 惊惧/呵斥/上报 = 真骨气，替掉现在的环境回避）。
- **一举三得**：吸收真演化 + 用反应顶回假的（修"软骨气"）+ 与结局裁决复用同一种判断。

### 4.3 导演升级为"有节制的世界引擎"（接问题①）
- 新权能：向仲裁**提交结构变更**；**主动推世界 / 温和把跑偏玩家拽向 crux**（推世界但不替玩家行动的"中间档"，区别于 railroad）。
- 这同时治了问题①（导演太被动）和问题②（世界不演化）——它俩本就同根。

## 5. 分期与优先级（结合"准备正式上线"）

| 期 | 做什么 | 价值 / 风险 | 上线相关 |
|---|---|---|---|
| **Phase 0** | 仲裁的**拒绝 + 反应**半边：玩家塞结构性反事实/越界 → 判"没挣到" → **NPC/导演真反应**（顶回去、造冲突），不再躲氛围。**不需要持久层。** | 直接补"世界有骨气、会反应"这个体验黑洞；又是后面一切的前置判断 | ✅ 唯一强相关 |
| **Phase 1** | 持久账本 + **剧本模式作者预定义结构里程碑**（甄嬛按作者设的条件上位）→ 写账本 → 叠回脊柱 | 让 authored 弧真能结构演化；**作者有界 = IP 安全** | 🟡 上线后第一个增量 |
| **Phase 2** | 自由模式 **LLM 仲裁判合法性 + 有界级联** = 真·活世界 | 风险最高（IP 漂移/失控）；**上 flag、缩范围、晚做** | ❌ 愿景，勿阻塞上线 |

**判断：**
- **别先追自由模式"完全活世界"**——登月、风险最大、最易把 IP 玩崩。
- **先做 Phase 0**：最小代价补体验黑洞，且是仲裁的拒绝半边（后面地基）。
- 始终记住**适度的"冻"是 IP 产品的特性**；目标是"**该认的认、该顶的顶**"，由仲裁一个闸来分，而非"什么都能改"。

## 7. 非目标（明确不做）

- 不追求"玩家说什么世界就变什么"（yes-man）。
- 不追求模拟全量级联涟漪；committed 事实只做**一阶效应**（名单移除/关系冻结/一次"世界反应"叙事）+ 让 NPC 从更新后的状态即兴反应。
- 不在上线前做 Phase 2（自由模式完全活世界）。
- 不打破现有 prefix-cache 经济性：账本变更低频，走快照叠加而非每 token 动态拼。
