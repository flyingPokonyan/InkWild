# InkWild World Scout（世界选品官）Skill 设计

> 状态：设计稿，待落地为 `~/.claude/skills/inkwild-world-scout/SKILL.md`
> 日期：2026-05-31
> 类型：可复用 skill（评判标准 + 选品方法论 + 剧本规划），供本人或其他 AI 长期使用

## 1. 目标与背景

InkWild 是「一个世界（舞台）挂多个剧本（剧情线）」的 AI 互动叙事引擎。当前只有 `wuyinzhen` 一个种子世界、一个剧本。要长期扩充内容库，需要一套**稳定的评判标准**，让 AI 能：

1. **自主联网找**「适配引擎 + 知名度高」的世界级 IP（如长安十二时辰、哈利波特），也能**反向生成**原创世界。
2. **验证世界是否真的适合** InkWild，能撑几个剧本。
3. 给出**剧本组合规划**：数量、先后顺序、各自精彩程度，附钩子提示。
4. 过线的产出**改编种子**，可直接喂 `/workshop` 创作工坊扩写。

这个 skill 就是那套标准 + 配套方法论。它**不替代**创作工坊（工坊吃一句 prompt 扩写完整世界/剧本），而是站在工坊**前面**，负责选品、验世、出高质量种子。

### 1.1 已锁定的决策

| 决策 | 结论 | 理由 |
|---|---|---|
| 选品轨道 | **双轨**：IP 搜判 + 原创反向生成 | 共享同一套评估内核；原创不是"找"是"按配方造" |
| 版权维度 | **完全不提** | 产品定位是创作工具（类剪映），用户自创内容，平台只提供工具 |
| 评判粒度 | 通用核心 5 维 + 薄题材透镜 | 避免"每题材一套 rubric"的维护地狱，又不漏题材坑 |
| 房屋风格 | 沿用 `design-spec-auditor.md` | 固定维度别膨胀 / 克制输出 / 反模式 / 关键原则 |
| 安装位置 | `~/.claude/skills/inkwild-world-scout/SKILL.md`（用户级）+ `docs/skill/` 镜像 | 用户级让其他 AI 也能用；镜像便于在 repo 内查阅 |

### 1.2 非目标（YAGNI）

- 不做法律/版权判定。
- 不做完整世界/剧本生成（那是创作工坊）。
- 不堆几十条候选——克制是硬约束。
- 不替代人的最终拍板——skill 给裁决 + 信心度，人决定做不做。
- **不重复 IP 复刻引擎的活**（见 §1.3）：不抓 IPKnowledgePack、不做世界层忠实复刻，那是工坊内已有系统的事。

### 1.3 与 IP 复刻引擎的边界（已存在的下游系统）

项目里已有 **IP 复刻引擎**（`services/ip_recognizer.py` + `ip_research_pipeline.py` + `ip_knowledge_packs` 表；spec 见 `2026-05-14-ip-fidelity-engine-design.md` + `...phase2-design.md`）。它活在**创作工坊的生成管线里**，职责是「**给定一个 IP，忠实复刻成世界**」：Stage 0 识别 `known_ip/hybrid/original` + 置信度 → strict 档用 Grok web_search + 百度百科 + 维基 + Tavily 抓取 → RAG 抽成结构化 `IPKnowledgePack`（角色/地点/势力/标志物/关键事件/称谓/时间线）→ 下游 14 stage 用 enum 硬约束角色地名忠于原作 → critic 卡 `must_have_characters` 命中率。其设计哲学是「**剧情走向不强约束**——底色对了，运行时演绎自然真实」。

边界划分（**互补，不重叠**）：

| | IP 复刻引擎（已存在，工坊**内**） | inkwild-world-scout（本 skill，工坊**前**） |
|---|---|---|
| 回答的问题 | "给定这个 IP，怎么忠实建出世界？" | "该选哪个 IP/世界？它适配引擎吗？能挂几个剧本？" |
| 输入 | 用户敲的描述（如"影视剧 逐玉"） | 一个方向/题材，或一个候选 IP |
| 输出 | 忠于原作的 world（角色/地名/势力到位） | 选品裁决 + 验世评分 + 剧本路线图 + 改编种子 |
| 剧情/谜题 | **明确不强约束** | **正是本 skill 补的洞**：谜题/线索链/结局/剧本组合 |

衔接：本 skill 选定 IP 后，产出物**喂进工坊** → 工坊触发 IP 复刻引擎建世界 → 再按本 skill 的剧本路线图逐个生成剧本。两个对齐原则：
1. **沿用引擎词汇**：`known_ip / hybrid / original` 三态 + `strict / loose / none` fidelity_mode；本 skill 给每个候选**推荐一个 fidelity_mode**。
2. **改编种子分工**：世界层的 IPKnowledgePack 交给引擎自动抓（本 skill **不重复抓**）；本 skill 重点产出**剧本层种子**（谜题/线索链/结局/视角）——引擎留空的部分。

## 2. Skill 脊柱

```
选品 ──┬─ IP 轨：WebSearch 扫面 → Grok 测热度(可选) → WebFetch 深读验证
       └─ 原创轨：把 5 维当配方反向构造世界种子
   ↓
验世（5 维适配评分，世界级 + 剧本级，分模式）
   ↓
排剧本（数接缝 → 评精彩度 → 定先后 → 出路线图）
   ↓
改编种子 + 推荐 fidelity_mode → 喂 /workshop
   · 世界层：工坊触发 IP 复刻引擎自动建世界（本 skill 不重复抓 IPKnowledgePack）
   · 剧本层：按剧本路线图逐个走 /workshop/generate/script（world_id + outline）
```

三阶段可单点切入（"帮我评估这个 IP" / "帮我排剧本"），也可一条龙。

## 3. 核心评判维度（5 条，钉死不膨胀）

从 `wuyinzhen` 种子逆向 + 成熟编剧手艺（见 §4）共同支撑。**每条在世界级和剧本级各看一遍，并分剧本/自由模式打分**——同一素材可能只适合一种模式。

| # | 维度 | 在问什么 | 不及格的样子 | 对应引擎系统 |
|---|---|---|---|---|
| 1 | **可隐藏的真相 / 未解张力** | 有没有能藏起来、能被玩家自然语言行动逐步逼出的核心？（剧本=真相；自由=悬而未决的冲突） | 人尽皆知的定论、结局没悬念、玩家无从下手 | 案件板 / 隐藏真相 / Director |
| 2 | **群像 + 冲突性私有信息** | 有没有一组人各揣冲突/互补的秘密，能切片分给不同 NPC？ | 单主角独角戏、只有一个知情人 | NPC secret+knowledge / 信息隔离 / 多 Agent |
| 3 | **有边界的舞台 + 日常肌理** | 有没有一个能被时钟驱动的有限舞台（地点有限、人物有作息、有生活）？ | 漂浮的宏大概念、没有"地方"、人物不过日子 | world_clock / NPC schedule / world_simulator |
| 4 | **多视角不对称切入** | 拆得出 2–4 个信息/能力不对称的可扮演身份吗？ | 只有一种合理视角，重玩没意义 | world_characters / 多角色开局 |
| 5 | **自带升级压力 / 可收束** | 有没有内生的倒计时或连环升级，能撑节奏和多结局？ | 静态、没有"再不动手就……"、无法自然收尾 | 倒计时 / 事件链 / endings |

打分用 `✅ 强 / 🟡 可 / ❌ 弱`，三个 ❌ 触碰红线（见 §8）。

> "玩家能否用自然语言行动撬动"作为 #1 的子测试，不单列——保持 5 条的克制。

## 4. 底层手艺库（已联网核对出处）

把成熟的写本手艺焊进 rubric，让标准站在真实方法论上而非单一种子的直觉。**以下出处经 Grok 联网核查（2026-05-31），均成立**，附精修。每条标明映射到 InkWild 哪个系统。

| 手艺 | 出处（已核对） | 精修 | 映射到 InkWild |
|---|---|---|---|
| **备情境，别备情节**（Don't prep plots, prep situations） | Justin Alexander，The Alexandrian 博客，2009 | 给"当前状态 + NPC 目标 + 工具"，不给预定事件序列 | 涌现式编排；自由模式 + intent_system |
| **三线索法则**（Three Clue Rule） | Justin Alexander，The Alexandrian，2008 | 每个关键结论 ≥3 条可达线索，防卡死 | 剧本线索链 + `rounds_without_progress` 保底 event |
| **公平推理**（Fair-play） | Knox 十诫（Ronald Knox, 1929）+ 范达因二十条（S.S. Van Dine, 1928） | 真相须从已铺线索可推；红鲱鱼要可解不可藏 | 隐藏真相 + 线索链可达性 |
| **每人一块秘密 · 信息不对称** | 起源 Mafia（Davidoff, ~1986）/狼人；现代精炼范例 Blood on the Clocktower（Steven Medway, ~2019） | BotC 定位为"现代范例"非起源 | NPC secret+knowledge 切片 = 信息隔离 |
| **人物要"想要什么"非只"知道什么"** | 斯坦尼斯拉夫斯基「objective」，《An Actor Prepares》1936 | objective（想达成什么）独立于 given circumstances（知道什么） | NPC intent（intent_system），不止 secret |
| **失败向前 · 节奏阀**（Fail forward） | PbtA / Apocalypse World（Vincent & Meguey Baker, 2010） | miss 触发复杂化推进剧情，不留死局 | `rounds_without_progress` 强制推进事件 |
| **为 AI 演绎而写：给情境留演绎空间** | Façade（Mateas & Stern, 2005），beat-based drama manager | 写"情境 + 动机 + beats"给 agent，不写死对白/分支树 | 多 Agent 编排吃情境，Narrator 织叙事 |
| **契诃夫之枪 + 红鲱鱼** | 契诃夫书信（~1889）；red herring 经 Cobbett 1807 普及 | 引入的元素须回收；误导要公平（与公平推理一致） | Director 铺设可回收细节 |
| **倒计时 · 升级赌注**（惊悚结构 / 长安十二时辰式 24h） | 类型片结构常识 | 时限逼迫行动、赌注逐幕升级 | world_clock 倒计时 + 多结局 |

落 SKILL.md 时每条会带 1 条 URL 出处（已收集，见本文档 §11）。

## 5. 题材透镜（薄、可无限扩展）

每题材一段话，只说三件事：**养分在哪 / 易踩的坑 / 改编要补什么**。新增题材 = 加一段，不重写核心。

- **悬疑 / 犯罪**：天生贴剧本模式（现成真相 + 线索链）。坑：过曝题材（开膛手杰克级）已被做烂。补：把"已知结论"重新藏起来，造信息隔离。
- **历史 / 半历史正剧（长安十二时辰这类）**：世界观厚、阵营冲突天然、倒计时易找。坑：**结局人尽皆知 = 没悬念**。补：聚焦"当时人不知道的那一刻"，或换成虚构小人物视角。
- **奇幻 IP（哈利波特这类）**：舞台规则清晰、群像现成、可玩角色多。坑：原著主线太强，玩家容易被"我知道剧情走向"剥夺代入。补：选原著缝隙里的支线/未写时段，造原创谜题。
- **神话 / 传说 / 志怪**：氛围强、留白多，贴自由模式。坑：纯氛围没钩子、玩家无事可做。补：在传说里埋一个可调查的具体谜。

剧本杀社区分类（本格/变格/情感/机制/恐怖，社区共识非单一作者，已核对）可作为题材透镜的补充参照系。

## 6. 双轨选品方法论

### 6.1 IP 轨（搜 + 判）

1. **定方向**：题材 / 时代 / 情绪基调（用户给，或 skill 建议）。
2. **扫面**：`WebSearch` 找候选 IP（按题材给 query 模板，如 `经典悬疑 IP 适合改编 沉浸式`、`X 题材 高人气 小说/剧/游戏`）。
3. **测热度（可选）**：`Grok`（端点见 §10）测 IP 当前热度/讨论度——它强在 X/实时。**仅用于热度判断**，token 极贵不高频用。
4. **深读验证**：`WebFetch` 拉候选 IP 的设定页（维基/Fandom/剧情简介），验证 §3 的 5 维结构是否真实存在。
5. **去重**：避开被影视游戏改烂的过曝 IP；优先"有料可查但没被做滥"的中等知名度。
6. **定 IP 性质 + 推荐 fidelity_mode**（对齐复刻引擎词汇）：
   - `known_ip`（明确指向某 IP）→ 推荐 `strict`（高复刻）或 `loose`（借鉴主线）。
   - `hybrid`（"赛博朋克版红楼梦"这类混搭）→ 推荐 `loose`。
   - 评分时标注：该 IP 用 strict 档复刻引擎能不能抓到足够素材（冷门新 IP 可能抓不全，提示降级 loose）。
7. **评分 → 排剧本**：过线进 §7。

### 6.2 原创轨（按配方反向造）

把 5 维当**生成配方**逐项构造（不是搜，是攒）：

1. **先定舞台**（维度 3）：一个有边界、能时钟驱动的地方 + 时代。
2. **再定群像秘密切片**（维度 2）：4–6 个人物，各揣冲突/互补的私有信息。
3. **再定可藏真相 / 核心张力**（维度 1）：能被行动逼出的核心。
4. **再定 2–4 个不对称视角**（维度 4）。
5. **再定倒计时 / 升级源**（维度 5）。
6. **自检 5 维** → 输出原创世界种子 + 剧本路线图 → 喂创作工坊。

原创轨对应复刻引擎的 `original` / `fidelity_mode=none`：工坊会识别为原创、跳过 IP Research，走自由创作。所以原创轨的世界种子要**自带足够纹理**（舞台/群像/规则），因为没有原作素材给引擎兜底。

## 7. 验世 + 剧本组合规划

把一个世界看成有多条**剧本接缝**（同一舞台 + 群像里，引擎能戏剧化的不同谜题/冲突）。

1. **数接缝**：列出这个世界能长出的候选剧本（每条 = 一个可藏真相/核心冲突）。
2. **评精彩度**：每条按 5 维打分。精彩度 = 点亮维度数 + 情感回报/赌注 + 新鲜度 + 真相清晰度。
3. **定先后**（典型三段）：
   - **入门本**：自包含、单子舞台、中等赌注，教玩家认识世界和几个关键 NPC，低前置知识。
   - **进阶本**：复用已熟悉的 NPC、加深其秘密、扩大舞台、抬高赌注，可假设玩家懂世界。
   - **收官本**：兑现世界核心张力，最高赌注，群像全员在场，结局分支最多。
4. **出路线图**：N 本，每本一句钩子 + 精彩度 + 推荐位次 + 为什么。

**示例（长安十二时辰）**：世界 = 盛唐长安 + 24h 倒计时 + 靖安司/各方势力。
- S1《坊市一桩命案》入门（单坊、自包含）→ S2《靖安司里的内鬼》进阶（政治、复用 NPC、秘密加深）→ S3《上元节袭击主线》收官（全员高赌注、多结局）。顺序按"对世界的熟悉度 + 赌注"升级。

## 9. 红线 / 反模式 / 关键原则

**一票否决红线**（命中即 ❌，不进短名单，只留引擎适配硬伤）：
- 无任何可藏的张力（维度 1 = ❌）
- 无人物群像（维度 2 = ❌）
- 无可落地的舞台（维度 3 = ❌）

**反模式**：
- 堆几十条不筛 → 用户被淹没。
- 抓过曝烂 IP → 没新鲜度。
- 只给结论不给来源链接 → 无法核实。
- 改编种子太空泛 → 喂不进创作工坊。
- 把"有名"当"好料" → 知名度 ≠ 引擎适配。
- 高频调 Grok 做小查询 → token 浪费（它单次百万级 prompt）。

**关键原则**：
1. **选品 ≠ 生成**：skill 出种子，创作工坊出成品。
2. **优先级 > 完整性**：5 个强候选 > 30 个泛候选。
3. **来源 > 论断**：IP 轨每条带 URL，原创轨标"原创"。
4. **适配 > 知名度**：5 维不过线，再有名也弃。
5. **手艺有出处**：rubric 的每条维度都能追到成熟方法论（§4）。

## 10. Grok 集成

- **形态**：OpenAI 兼容 `POST {GROK_BASE_URL}/v1/chat/completions`，`model=grok-4.20-multi-agent-xhigh`，`Authorization: Bearer {GROK_API_KEY}`。
- **凭据**：**不硬编码进 skill**。SKILL.md 让 AI 读环境变量 `GROK_BASE_URL` / `GROK_API_KEY`；用户已在本会话提供可用端点用于测试。
- **特性（实测 2026-05-31）**：带联网、引用质量高；单次 prompt 灌 120–155 万 token，偏慢但**成本不设限**（用户明确）。
- **用途定位**：① IP 实时热度/讨论度判断；② 需要带出处的深度核查。放开用，不必为省 token 回避。
- **后端已有 Grok**：IP 复刻引擎的 IP Research 已把 Grok web_search 当主搜索源（`backend/llm/grok.py` 已实装，grok-4.20 系列）。本 skill 的 Grok 是 **agent 层 curl 调用**，与后端 provider 并行、互不影响。
- **兜底**：Grok 不可用时，IP 轨降级到纯 `WebSearch` + `WebFetch`，流程不中断。
