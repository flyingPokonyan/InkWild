# 《甄嬛传》忠实世界 + 章节本 — 计划与代办

> 日期：2026-05-31　目标：打造一个**完美、知名度高、忠实原著**的甄嬛传世界，挂一长串剧本。
> 用 `inkwild-world-scout` 选品验世 + 工坊 IP 复刻引擎建世界。研究底稿：Grok 联网核对（76集剧集+原著），出处见维基甄嬛传角色列表等。

## 0. 设计原则（关键，所有本共用）

1. **忠实原著、不魔改**：事件/人物/时间线照剧走。剧本是"跟着剧情走的章节本"，不是"我藏个谜让你猜"。
2. **剧透无关**：乐趣 = ①**戏剧性反讽**（你知道皇后害纯元、安陵容会害眉庄，但你扮的角色身在局中）+ ②**能动性分歧**（你能做跟电视不一样的动作）。
3. **双结局模板**：每个本至少给 **复刻原著结局**（你走了和原著一样的路）+ 若干 **改写命运结局**（救下眉庄 / 避开纯元旧衣陷阱 / 提前扳倒华妃 / 自己翻车……）。
4. **自由模式才是真沉浸**：忠实后宫里随便扮、随便改，做电视没发生的事。剧本模式 = 引导章节。
5. **原著"秘密"照留**：欢宜香、害纯元等只用来驱动 NPC 行为（信息隔离），不当玩家必破的谜。

自由模式才是"做电视没发生的事"的真沉浸。推荐起点角色（靠 world_characters，不写剧本）：
- **新晋小主**（碎玉轩起点）——白板代入，最自由

- **太医**（温实初式）——能验毒、各宫游走
- **失势隐忍妃**（端妃/敬妃式）——低调布局

## 4. 喂工坊流程 + IP 复刻度怎么保证

### 复刻度机制（代码实证）

- **世界复刻**是引擎工程化保证的，**不靠描述 prose**：`ip_recognizer` 检测 known_ip → `ip_research_pipeline` 抓 `IPKnowledgePack` → 下游 enum 硬约束 → critic 卡 `must_have` 命中率。输入只要点到"甄嬛传"+ strict。§1 必到清单是**验收对照**，非机制。
- **剧本自动继承世界的 IP pack**：`_build_script_world_data` 按 world_id 取 `ip_knowledge_packs` → `create_script` 继承 `fidelity_mode` → `events_data_builder` 注入【强约束·原作锚点】（必备角色）。→ 人物/地点复刻自动。
- **但 IP pack 不含剧情时间线**，引擎对"剧情走向"不强约束 → **"跟着剧情走"全靠 outline**。

### 三层保险（保证而非指望）

| 层 | 做法 | 谁保证 |
|---|---|---|
| a 人物/地点 | 世界 strict 生成 → 剧本自动继承 pack 锚定 | 引擎（自动） |
| b 剧情线 | outline = **结构化真实事件链 + 双结局 + 视角**（非泛描述） | 喂的 outline |
| c 锁稿 | 生成后在草稿里人工校 `events_data`/`endings_data`，把名场面触发+结局钉死再发布 | 人工兜底 |

### 步骤

1. **世界**：`/workshop/generate/world`，输入"电视剧《甄嬛传》"+ strict；产出草稿 → 按§1 清单校 → **发布**（发布才 backfill `pack.world_id`，剧本才能继承）。
2. **每个剧本**：`/workshop/generate/script`，绑定已发布的甄嬛传 world_id；outline = 代办里该本的「弧线 + 关键事件序列（events 骨架）+ 复刻原著结局 + 改写命运结局（endings）+ 推荐视角」。
3. **锁稿**：草稿里校对名场面（滴血验亲、纯元旧衣等）的触发条件与结局分支，确认无误再发布。

> ⚠️ 前提：世界必须经工坊 strict 生成并发布。手搓/老世界无 pack → 剧本退化到弱 `ip_canon`，复刻变差。

## 5. 待确认 / 可扩展

- 大本与小本事件是否共享同一套 events（避免重复）：建议大本走完整事件链，小本只取其中一段触发条件。

- 长尾小本（S15+）按需开。
