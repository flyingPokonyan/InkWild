## ImageGenerator API 适配

- `generate_image(prompt, aspect_ratio=...)` returns `ImageResult` (not a URL string)
- `save_generated_image_result(storage, result, key)` handles URL-type and bytes-type results, returns stored URL string
- `IMAGE_PLACEHOLDER_URL` constant imported from `services.image_storage`
- `get_image_storage()` called lazily inside `_run_images` (only when `image_gen` is set)

## Implementation

`_run_images` in `WorldCreatorAgentV2`:
1. **Fast path**: if `self.image_gen` is falsy, assign `IMAGE_PLACEHOLDER_URL` to all image fields and return immediately — no I/O
2. **Real path**: build 3 main image tasks (cover 16:9, hero 16:9, poster 3:4) + 1 per `is_image_target=True` character (1:1 portrait)
3. `asyncio.Semaphore(6)` caps concurrent Seedream calls (spec §12.3)
4. `asyncio.gather(*coros, return_exceptions=True)` runs all tasks concurrently
5. Single-task failure → `IMAGE_PLACEHOLDER_URL` fallback, pipeline continues
6. Emits `subtask_started` (batch kickoff) + `subtask_completed` per task (spec §8.3)
7. Emits stage-level `started` + `completed` with `payload_summary`
