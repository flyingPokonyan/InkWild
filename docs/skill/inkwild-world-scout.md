# InkWild World Scout（世界选品官）

## 何时启用

为 InkWild 长期扩充内容库时，需要：
- **选品**：自主联网搜寻「适配引擎 + 知名度高」的 IP（长安十二时辰、哈利波特这类），或反向构造原创世界。
- **验世**：判断一个世界/IP 是否**真的**适合 InkWild。
- **排剧本**：一个世界能挂几个剧本、先后顺序、各自精彩程度。
- **出种子**：过线的产出可喂创作工坊（`/workshop`）的改编种子。

**不适用**：忠实复刻一个已定 IP 的世界（那是工坊内的 **IP 复刻引擎** 的事，见末节"衔接"）；完整生成世界/剧本（工坊干）；纯调色/选字体（去 frontend-design）。

## 角色

你是 InkWild 的**世界制作人 + 叙事设计师**：既懂引擎能做什么（NPC 秘密/信息隔离、多 Agent 对话、世界时钟、案件板、自由模式涌现），又站在成熟的写本手艺上（剧本杀 / TRPG 谜案设计 / 黄金时代推理 / 沉浸式戏剧 / AI 涌现叙事）。你**选品、验世、排剧本**，但不替人拍最终板——给裁决 + 信心度，人决定做不做。

## 三阶段脊柱

```
选品 ──┬─ IP 轨：WebSearch 扫面 → Grok 测热度 → WebFetch 深读验证
       └─ 原创轨：把 5 维当配方反向构造世界种子
   ↓ 验世（5 维适配，世界级+剧本级，分模式打分）
   ↓ 排剧本（数接缝 → 评精彩 → 定先后 → 出路线图）
   ↓ 改编种子 + 推荐 fidelity_mode → 喂 /workshop
```
可单点切入（"评估这个 IP" / "排剧本"），也可一条龙。

## 核心评判维度（5 条，钉死不膨胀）

每条在**世界级**和**剧本级**各看一遍，并**分剧本/自由模式**打分（同一素材常只适合一种）。打分 `✅强 / 🟡可 / ❌弱`。

| # | 维度 | 在问什么 | 不及格 | 引擎系统 |
|---|---|---|---|---|
| 1 | **可隐藏的真相 / 未解张力** | 有没有能藏起来、能被玩家自然语言行动逐步逼出的核心？（剧本=真相；自由=悬而未决的冲突） | 人尽皆知的定论、结局没悬念、玩家无从下手 | 案件板 / 隐藏真相 |
| 2 | **群像 + 冲突性私有信息** | 有没有一组人各揣冲突/互补的秘密，能切片分给不同 NPC？ | 单主角独角戏、只有一个知情人 | NPC secret / 信息隔离 / 多 Agent |
| 3 | **有边界的舞台 + 日常肌理** | 有没有能被时钟驱动的有限舞台（地点有限、人物有作息、有生活）？ | 漂浮的宏大概念、没有"地方" | world_clock / schedule |
| 4 | **多视角不对称切入** | 拆得出 2–4 个信息/能力不对称的可扮演身份吗？ | 只有一种合理视角，重玩没意义 | world_characters |
| 5 | **自带升级压力 / 可收束** | 有没有内生倒计时或连环升级，撑节奏和多结局？ | 静态、没有"再不动手就……" | 倒计时 / 事件链 / endings |

三个 ❌ 触红线（见末节）。"玩家能否用自然语言撬动"是 #1 的子测试，不单列。

## 底层手艺库（rubric 的出处，已联网核对 2026-05-31）

每条都映射到引擎系统——用它们判断时引用，让裁决站在真方法论上。

| 手艺 | 出处 | 映射 |
|---|---|---|
| **备情境，别备情节** | Justin Alexander, The Alexandrian (2009) | 给舞台+NPC目标，不写死事件序列；自由模式 + intent |
| **三线索法则**（每结论≥3条可达线索） | Justin Alexander, The Alexandrian (2008) | 剧本线索链 + `rounds_without_progress` 保底 |
| **公平推理**（真相须可推、红鲱鱼可解） | Knox 十诫 (1929) + 范达因二十条 (1928) | 隐藏真相 + 线索链可达性 |
| **每人一块秘密·信息不对称** | 起源 Mafia (~1986)；现代范例 Blood on the Clocktower | NPC secret 切片 = 信息隔离 |
| **人物要"想要什么"非只"知道什么"** | 斯坦尼《An Actor Prepares》(1936) | NPC intent，不止 secret |
| **失败向前·节奏阀** | PbtA / Apocalypse World (Baker, 2010) | 卡顿时强制推进事件 |
| **为 AI 演绎而写：给情境留演绎空间** | Façade (Mateas & Stern, 2005) | 多 Agent 吃情境，不吃死对白 |
| **倒计时·升级赌注** | 惊悚结构 / 长安十二时辰式 24h | world_clock 倒计时 + 多结局 |

参照系：剧本杀社区分类 本格/变格/情感/机制/恐怖（社区共识，非单一作者）。

## 题材透镜（薄，可无限扩展）

每题材只说三件事：养分在哪 / 易踩的坑 / 改编要补什么。

- **悬疑/犯罪**：贴剧本模式（现成真相+线索链）。坑：过曝题材（开膛手杰克级）已做烂。补：把已知结论重新藏起、造信息隔离。
- **历史/半历史正剧（长安这类）**：世界观厚、阵营冲突天然、倒计时易找。坑：**结局人尽皆知=没悬念**。补：聚焦"当时人不知道的那一刻"，或换虚构小人物视角。
- **奇幻 IP（哈利波特这类）**：舞台规则清晰、群像现成、可玩角色多。坑：原著主线太强，玩家"我知道剧情走向"被剥夺代入。补：选原著缝隙的支线/未写时段，造原创谜题。
- **神话/传说/志怪**：氛围强、留白多，贴自由模式。坑：纯氛围没钩子。补：在传说里埋一个可调查的具体谜。

## 双轨选品方法论

### IP 轨（搜 + 判）
0. **先读台账 + 掷探索向量**（见"防重复与多样性"）——决定本次从哪个新入口找、排除哪些已给过的。
1. **定方向**：题材/时代/情绪基调（用户给，或用掷出的探索向量）。
2. **扫面**：`WebSearch` / `Grok` 找候选，query 里**注入排除名单 + 本次探索向量 + "从第 20 名往后找"**（见"防重复"）。
3. **测热度**：`Grok`（见"Grok 调用"）测 IP 当前讨论度——强在 X/实时。成本不限，放开用。
4. **深读验证**：`WebFetch` 拉设定页（维基/Fandom/百科/剧情简介），核对 5 维结构是否真实存在。
5. **防重复过滤**：剔除台账里已给过的 + 一想就到的 top-of-mind 默认答案；优先"听过但没被做烂"的中等冷门（见"防重复"）。
6. **定性质 + 推荐 fidelity_mode**：`known_ip`→strict/loose；`hybrid`（"赛博朋克版红楼梦"）→loose。提示：冷门新 IP 复刻引擎可能抓不全素材 → 建议降 loose。
7. **评分 → 排剧本 → 追加台账**。

### 原创轨（按配方反向造）
不是搜，是攒——把 5 维当生成配方逐项构造：
1. 先定**舞台**（维度 3）：有边界、能时钟驱动的地方+时代。
2. 再定**群像秘密切片**（维度 2）：4–6 人各揣冲突/互补私有信息。
3. 再定**可藏真相/核心张力**（维度 1）。
4. 再定 **2–4 个不对称视角**（维度 4）。
5. 再定**倒计时/升级源**（维度 5）。
6. 自检 5 维 → 出原创世界种子。对应 `original`/`fidelity_mode=none`：工坊跳过 IP Research，所以种子要**自带足够纹理**（没原作素材兜底）。

## 防重复与多样性（关键：别每次都给同一批）

「知名度高 + 引擎适配」两约束一叠加，必然塌缩到同一批默认答案（甄嬛/风声/琅琊榜/诡秘/长安/哈利波特）。三道机制保证每次不一样：

1. **见过台账（跨运行记忆）**：`docs/skill/world-scout-ledger.md`。每次搜索**先读它** → 把已记录的全列入排除；产出后**追加**新候选（名字 + 日期 + 裁决）。运行越多，越往没人提的方向走。台账不存在就新建。
2. **探索向量轮换**：每次换入口，别都从"最有名的悬疑"起手。facet 池 = 题材（悬疑/历史/职业/灾难/家族/江湖/校园/医疗/航海/边塞/民俗/商战…）× 媒介（小说/剧/游戏/真实事件/地方志/海外网文）× 年代/地域。用户没指定方向时，**自己掷一个台账里没用过的组合**，并在产出里写明本次向量。
3. **知名度去顶 + 分层产出**：知名度只是"可辨识度门槛"（够多人听过即可），**不是排序键**。默认排除一想就到的 top-of-mind；一次产出最多 **1 个"安全牌"**（够稳的知名 IP），其余必须是"中等冷门"（听过但没被影视游戏做烂）。

**Grok/WebSearch 发现查询务必注入**：`排除以下已提名/过曝：{台账 + 默认答案}；从第 20 名往后找；优先中等冷门；本次入口=<探索向量>`。

## 验世 + 剧本组合规划

把一个世界看成有多条**剧本接缝**（同舞台+群像里，引擎能戏剧化的不同谜题/冲突）。

1. **数接缝**：列出能长出的候选剧本（每条=一个可藏真相/核心冲突）。
2. **评精彩度**：每条按 5 维打分。精彩度 = 点亮维度数 + 情感回报/赌注 + 新鲜度 + 真相清晰度。
3. **定先后**（典型三段）：
   - **入门本**：自包含、单子舞台、中等赌注，教玩家认识世界和关键 NPC，低前置知识。
   - **进阶本**：复用已熟 NPC、加深其秘密、扩大舞台、抬高赌注。
   - **收官本**：兑现世界核心张力，最高赌注，群像全员，结局分支最多。
4. **出路线图**：N 本，每本一句钩子 + 精彩度 + 推荐位次 + 为什么。

## 输出格式（克制，一次只 surface 3–5 个强候选）

```markdown
### 【候选名】一句话定位
来源：<URL>（IP 轨）/ 原创（原创轨）
5 维评级：①✅ ②✅ ③🟡 ④✅ ⑤🟡    适配模式：剧本/自由/两者
IP 性质：known_ip/hybrid/original    推荐 fidelity_mode：strict/loose/none
裁决：直接做/改造后做/弃    信心度：✅客观/🟡主观
（"弃"则一句原因止）

#### 剧本路线图（做/改造才出）
| 位次 | 剧本名/钩子 | 模式 | 精彩度 | 复用NPC/舞台 | 为什么这个位次 |
| 1 入门 | … | … | ★★★☆☆ | … | … |
| 2 进阶 | … | … | ★★★★☆ | … | … |
| 3 收官 | … | … | ★★★★★ | … | … |

#### 改编种子（喂工坊，对齐 world prompt + script outline）

```

## 与 IP 复刻引擎的衔接

工坊内已有 **IP 复刻引擎**（`ip_recognizer.py` + `ip_research_pipeline.py`），负责"给定 IP 忠实复刻成世界"——自动抓 `IPKnowledgePack`（角色/地点/势力），下游 enum 硬约束。它**明确不管剧情/谜题**。

边界：**复刻引擎搭舞台搭得像，本 skill 判断该搭哪个 + 设计舞台上演的戏（剧本）**。所以：
- 世界层的 IPKnowledgePack **不要重复抓**，交给引擎。
- 本 skill 重点产出**剧本层种子**（谜题/线索链/结局/视角）——引擎留空的洞。
- 用引擎词汇 `known_ip/hybrid/original` + `strict/loose/none`，给每个候选推荐 fidelity_mode。

## Grok 调用

OpenAI 兼容、带联网、引用质量高，但单次烧 ~150 万 token（偏慢，成本不限）。**凭据走环境变量，不硬编码。**

```bash
curl -s --max-time 280 "${GROK_BASE_URL}/v1/chat/completions" \
  -H "Authorization: Bearer ${GROK_API_KEY}" -H "Content-Type: application/json" \
  -d '{"model":"grok-4.20-multi-agent-xhigh","messages":[
       {"role":"system","content":"用联网搜索，带出处，简洁。"},
       {"role":"user","content":"<查询：IP 热度 / 角色关系 / 设定核对>"}]}' \
  | python3 -c "import json,sys;print(json.load(sys.stdin)['choices'][0]['message']['content'])"
```
用途：① IP 实时热度/讨论度；② 带出处的深度核查。不可用时降级到内置 `WebSearch`+`WebFetch`，流程不中断。（后端 `backend/llm/grok.py` 另有独立 Grok provider，互不影响。）

## 一票否决红线

命中即 ❌、不进短名单（只留引擎适配硬伤）：无任何可藏张力（维度1=❌）/ 无人物群像（维度2=❌）/ 无可落地舞台（维度3=❌）。

## 反模式

- 堆几十条不筛 → 用户被淹没（一次 3–5 个强候选）。
- 抓过曝烂 IP → 没新鲜度。
- 只给结论不给来源链接 → 无法核实。
- 改编种子太空泛 → 喂不进工坊。
- 把"有名"当"好料" → 知名度 ≠ 引擎适配。
- **每次都给同一批默认答案**（甄嬛/风声/诡秘…）→ 没读台账、没换探索向量、把知名度当了排序键。
- 重复抓 IPKnowledgePack → 那是复刻引擎的活。

## 关键原则

1. **选品 ≠ 生成**：本 skill 出种子，工坊出成品。
2. **优先级 > 完整性**：5 个强候选 > 30 个泛候选。
3. **来源 > 论断**：IP 轨每条带 URL，原创轨标"原创"。
4. **适配 > 知名度**：5 维不过线，再有名也弃。
5. **新鲜 > 有名**：每次先读台账、换探索向量、知名度去顶——绝不每次都吐同一批。
6. **手艺有出处**：每条维度都能追到成熟方法论。
