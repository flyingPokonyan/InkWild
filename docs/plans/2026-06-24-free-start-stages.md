# 自由模式「人生进度」起点选择（free_start_stages）

> 状态：spec / 待实现 · 2026-06-24
> 作者：jie + Claude（设计讨论沉淀）

## 0. 一句话

给自由模式加一个**可选的起点选择**：玩家进世界、选完主角后，可以挑「从角色人生的哪一段开始」（炼气期 / 元婴期、常在 / 熹贵妃 …），而不是永远从同一个写死的初始状态开局。纯加法，不动引擎核心。

---

## 1. 背景 / 要解决的问题

- 长篇 IP（凡人修仙传跨炼气→飞升、甄嬛传跨选秀→太后）被压成**一个静态世界**，没有「现在演到哪」的概念。
- 自由模式每局 = 同一张固定初始快照（`第1天·上午` + `character.initial_location` + 全新关系），**无法选择从故事哪一段开始**。
- 结果：玩主角想体验后段、或嫌前期养成慢的玩家**没有出路**。
- 剧本模式靠 `script_setting` + events 天然有「入口」，但**不能逼自由模式玩家去玩剧本**。

→ 缺的不是「时间线引擎」，是一个轻量的**起点预设**。

## 2. 设计原则（已拍板）

1. **纯加法、向后兼容**：新增可选字段；为空 = 走今天的老行为。已有世界 / 全部剧本 / 剧本模式不受影响。
2. **引擎核心零改动**：orchestrator / director / npc / 事件系统一行不改。起点 = 「换一张初始 GameState 快照 + 一段开场定调」，是**开局前装配**，不是模拟逻辑。
3. **标题用「进度里程碑」，地点退到后台**：玩家看到的是 `元婴期` / `熹贵妃`（角色人生进度），不是地点。进度轴**按题材由 AI 用 IP 原生体系命名**（修仙=境界、宫斗=位分、武侠=江湖地位…）。地点仍需存（引擎要落点），但不是卖点、不当标题。
4. **只给主角配整套阶段**：跨纪元的人生进度只有主角才有；配角各归其所属纪元（用其 `initial_location`，不显起点步）。
5. **文案去 AI 味**：具体 > 华丽，把事说清 > 烘托情绪，禁成语堆砌。写进生成 prompt 当硬约束。

## 3. 数据模型（加法）

世界上新增一个可选字段：

```python
# models/world.py — World
free_start_stages: Mapped[dict | None] = mapped_column(_JSONB, nullable=True, default=None)
```

结构：

```jsonc
{
  "protagonist_character_id": "<world_character.id>",  // 这套阶段属于谁（其人生弧=整部传）
  "stages": [
    {
      "id": "stage_yuanying",
      "milestone": "元婴期",                 // 进度里程碑（题材原生轴）—— 行标题主体
      "subtitle": "乱星海大修士",            // 里程碑的情境定位 —— 与 milestone 同行
      "tagline": "爽感流，开局就有分量",       // 节奏取向 —— 行副标，一行
      "order": 3,                            // 早→晚排序
      "era": "人界·乱星海时期",               // 对齐 shared_events 的纪元（生成时溯源用）
      "start_location": "乱星海",            // 覆盖 GameState.current_location（后台字段）
      "opening_framing": "你已经是元婴修士，在乱星海熬了几百年，立住了脚。星宫、逆星盟都不敢轻易动你。元瑶跟你一道走，这片海域你说话有人听。",
      "known_relations": [                   // 可选：开局就「认识谁」，顺手解决陌生人/旧识错位
        {"npc": "元瑶", "standing": "同行"},
        {"npc": "极阴祖师", "standing": "对头"}
      ]
    }
  ]
}
```

**主角识别**：`narrative_weight` 不够用（凡人韩立、南宫婉都是 100）。故由**生成阶段显式产出** `protagonist_character_id`，不靠运行时推断。

> 迁移：一条 Alembic 加列。其余表 / 模型不动。

## 4. 生成管线（+1 阶段）

- v2 pipeline 在 **`shared_events` 之后**加一个新阶段 `free_start_stages`（依赖纪元骨架）。
- 输入：已生成的 `shared_events`（带 `era`）+ `base_setting` + 主角的 `WorldCharacter`。
- 输出：3–4 档，覆盖早 / 中 / 后（+ 可选终盘），每档按上面的结构。
- **Prompt 要点**：
  - 用**这个 IP 原生的进度体系**给 `milestone` 命名（修仙=境界 / 宫斗=位分 / …），别自造。
  - **`tagline` 必须自带「体验类型 · 一句感受」**，格式如「养成流 · 从最底层的新人熬起」「爽感流 · 开局即一方大修士」「终局 · 登顶之后的空与静」。理由：移动端没有 hover 出来的处境卡，玩家只能靠行内的 `milestone·subtitle`（标题）+ `tagline`（描述）做判断——所以 tagline 必须同时说清**走到哪一段**和**选了是什么玩法体验**，不能只写场景。`milestone` 是 IP 黑话（代入感），`tagline` 兜清晰度。
  - `opening_framing` **去 AI 味硬约束**：1–2 句、第二人称、说清「你此刻是谁 / 刚发生什么 / 身边有谁」，禁四字成语连发、禁华丽辞藻。
  - `known_relations` 从该纪元在场的 NPC 里挑 2–3 个。
  - `start_location` 取该纪元主角的合理落点。
- **成本**：纯文本、不生图，约 3–5K token / 世界，单世界几分钱，占世界总生成成本 <1%。
- **存量 backfill**：凡人 + 甄嬛各跑一次本阶段（复用现成 `shared_events`），不重抽整世界。

## 5. 运行时（start_game，集中一处）

`services/game_service.py::start_game`：

- 新增可选参数 `start_stage_id: str | None`（**仅自由模式**有意义）。
- 解析对应 stage 后，对 `initial_state` 做覆盖：
  - `current_location` ← `stage.start_location`（同时 `visited_locations` 跟着改）。
  - `opening_prompt` 用 `stage.opening_framing` **替换**现有「刚刚抵达 X」那句（不能并存——「刚刚抵达七玄门」与「你已是元婴修士」会自相矛盾）。无 stage 时仍用旧的「刚刚抵达」。
  - `npc_relations`：按 `known_relations` 给对应 NPC 抬 trust / 标 standing（在 `infer_initial_stances` 之上叠加，或对这些 NPC 跳过 stance 推断直接用预设）。
- **向后兼容**：`start_stage_id` 为空，或世界没有 `free_start_stages`，或选的不是主角 → 完全走今天的逻辑。

> 引擎核心、事件 DSL、SSE 协议都不碰。

## 6. 前端（start 页加一步，复用现成组件）

文件：`app/worlds/[id]/start/page.tsx`（现成分步向导）。

- **新增 `stage` 步**：
  - 仅 `mode === "free"` 且 `world.free_start_stages` 非空 **且选中角色 == 主角** 时出现。
  - 步序：`mode → character → stage → confirm`（`getSteps` + reducer 加这一步）。
  - 步首加一句**概念说明行**「想从{主角名}人生的哪一段开始？」——点明这一步是在选「角色人生的某个阶段作为开场」，否则玩家可能不知道 `常在`/`元婴期` 这种里程碑代表"起点选择"。
- **选项 = 复用「模式选择」那套 `ListChoiceOption` 行式**，**不用** 剧本的 `MediaChoiceCard` 封面卡。行放 `milestone · subtitle`（标题）+ `tagline`（副标，一行）。
- **响应式两栏（落地版）**：
  - **桌面（≥768px）**：`grid` 左列表（440px）/ 右处境（1fr），细竖线（`border-left`）分隔，整组 `max-width 880` 居中。右处境是当前 hover/选中那一档的常驻读区：serif 小标题 + `落点`/`认识的人`（caps 元数据）+ `opening_framing`（叙事字体，不截断）。解决"窄列飘在宽屏黑里"。
  - **移动端（<768px）**：`.lv-stage-detail` **`display:none`**，**只显行**。选择信息靠行内 `milestone·subtitle + tagline` 已足够；处境块在移动端无 hover 只会显第一档，是摆设，故隐藏。
  - 不用 `CardDetailStrip`（那是给媒体卡 carousel 的居中条）；处境读区是 stage 步内联的轻量块（无填充底，仅竖线分隔）。

字段 → 落点：

| 预设字段 | 落点 |
|---|---|
| `milestone` + `subtitle` | `ListChoiceOption.title` + 桌面右栏 serif 小标题 |
| `tagline` | `ListChoiceOption.description`（一行，移动端唯一的体验说明） |
| `opening_framing` | 桌面右栏正文（叙事字体）；移动端不显 |
| `start_location` | 桌面右栏「落点」元数据；移动端不显 |
| `known_relations` | 桌面右栏「认识的人」元数据；移动端不显 |

布局：


- `handleStart` 多带 `start_stage_id`；`startGame`（`stores/game.ts` + `lib/api`）透传到后端。
- i18n：步骤标题「起点」等进 `i18n/zh.json` / `en.json`。
- **API 依赖（别漏）**：世界详情响应（`schemas/` + worlds 路由序列化）要**新增暴露 `free_start_stages`（含 `protagonist_character_id`）**，否则前端无法判断「该不该出起点步」「选中的是不是主角」。这是额外但必要的一处加法。

## 7. 内容样例（去 AI 味 · 已定稿）

### 凡人修仙传（主角：韩立）
- **炼气期 · 七玄门少年**｜养成流 · 从凡人少年白手起步
  > 你是七玄门的普通弟子，资质一般，手里刚多了个来历不明的小瓶。师父墨大夫对你不错，可你慢慢觉得他教的「长春功」哪儿不对劲。修仙才刚开头，你还什么都不是。
  > 落点：七玄门 · 认识：墨大夫（待你和善、另有盘算）、厉飞雨、张铁
- **筑基期 · 黄枫谷弟子**｜闯荡期 · 入了门派，初露锋芒
  > 你进了黄枫谷，在一场试炼里拼下来，筑了基。年轻一辈里有了点名气，也跟王蝉那种人结了死仇。手里有了底子，麻烦也跟着来。
  > 落点：黄枫谷 · 认识：南宫婉（试炼时打过照面）、死对头王蝉
- **元婴期 · 乱星海大修士**｜爽感流 · 开局即一方大修士
  > 你已经是元婴修士，在乱星海熬了几百年，立住了脚。星宫、逆星盟都不敢轻易动你。元瑶跟你一道走，这片海域你说话有人听。
  > 落点：乱星海 · 认识：元瑶（同行）、对头极阴祖师、六道极圣
- **大乘期 · 灵界老怪**｜终盘 · 修为封顶，飞升在即
  > 你飞升到了灵界，打完万族那一仗，修为到了大乘。紫灵、冰凤跟你结盟，百族的盟约压在你身上。下一步是飞升仙界，可那边的事比这边更难。
  > 落点：灵界 · 认识：紫灵、冰凤（结盟）、古魔一系的敌人

### 甄嬛传（主角：甄嬛）
- **常在 · 初进碎玉轩**｜养成流 · 从最底层的新人熬起
  > 你刚选秀进宫，被安排在偏远的碎玉轩。皇上还没见过你，华妃当权，皇后面上和气。你没靠山，每一步都得自己掂量。
  > 落点：碎玉轩 · 认识：眉庄、安陵容（一起进宫的姐妹）、华妃（看新人不顺眼）
- **妃位 · 盛宠之时**｜争锋期 · 盛宠加身，也成了靶子
  > 你得了皇上的宠，也成了华妃的眼中钉。眉庄被人算计假孕，欢宜香的事还没揭开，翊坤宫和你这边已经摆开了。这时候，天真要不得。
  > 落点：养心殿一带 · 认识：温实初、眉庄（帮你）、华妃当道、皇后在暗处使劲
- **熹贵妃 · 回宫之后**｜权谋巅峰 · 这一次换你执子
  > 甘露寺那段过去了，你以熹贵妃的身份回了宫。这一回轮到你布局——滴血验亲、扳倒皇后，你不再是任人摆弄的小主。
  > 落点：永寿宫 · 认识：槿汐、苏培盛（心腹）、皇后（最后的对手）、果郡王（放不下的人）
- **（可选）太后 · 登顶之后**｜终局 · 登顶之后的空与静
  > 你做了太后，后宫到头了。当年的对手一个不剩，这地方也没剩几个能说话的人。想要的都拿到了，然后呢？
  > 落点：寿康宫 · 认识：故人多已不在；新帝和朝堂是另一盘棋

## 8. 明确不做

- **不做数值进度系统**：「元婴 / 贵妃」是叙事层建立（开场定调 + 阵容 + LLM 维持），引擎不存修为/位分数值。这是现架构天花板，本方案不打破。
- **不做存档继承 / NG+**：玩家靠「选起点」自由切入口，而非攒一个长期号。
- **不改剧本模式**。

## 9. Scope

| 层 | 工作量 | 内容 |
|---|---|---|
| 数据模型 | 小 | `World.free_start_stages` + 1 条 Alembic |
| 生成 | 中 | pipeline +1 阶段 + prompt（去 AI 味 / 原生进度轴） |
| 存量 backfill | 小 | 凡人 + 甄嬛各跑一次本阶段 |
| 运行时 | 小 | `start_game` 加 `start_stage_id` + 应用预设（1 处） |
| 前端 | 中 | start 页加 `stage` 步：`ListChoiceOption` 行 + 响应式两栏（桌面右栏处境 / 移动端只显行） |

**引擎核心：0 改动。**

## 10. 实现分期（review 后定）

**v1（先做，不碰生成）**：手搓凡人 + 甄嬛两套 `free_start_stages`（§7 文案已定稿）直接落库 → 后端 model 字段 + 迁移 + `start_game` 应用 + API 暴露 → 前端 `stage` 步 → 真机点。**全程不碰生成管线、不碰引擎核心**，最快看到效果。

**v2（手感验证后再做）**：把 §4 的生成阶段补进 v2 pipeline，让新世界自动产出起点。

## 11. 三个 open question — 已定默认（除非另议）

- **主角识别** → 由数据显式给 `protagonist_character_id`（v1 手填，v2 生成产出）；**不**运行时按 `narrative_weight` 猜（凡人韩立/南宫婉都是 100，猜不准）。
- **多主角** → MVP **单主角**（一个世界一套阶段）。
- **`known_relations` 规则** → 命中的 NPC **跳过 `infer_initial_stances`，直接用预设**（trust 取一个中高默认，如 6/10；standing 写进 note）；未命中的 NPC 照旧走 stance 推断。
