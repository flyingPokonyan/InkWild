# Product

## Register

brand

## Users

两类用户，地位并重：

- **玩家 / 读者** — 移动端为主，来沉浸式地玩 AI 互动叙事。两种玩法：剧本模式（预设谜题 / 真相 / 结局，像在解一部推理剧）和自由模式（纯开放世界）。他们要的是「被一个故事接住」：低门槛进入、沉浸、有代入感。
- **创作者** — 用创作工坊（AI 生成世界 + 剧本，含联网检索、AI 配图、草稿 / 发布流程）造内容。他们要的是「我的想法能快速变成能玩的世界」。

营销首页 / 发现页要同时对这两类人说话：先让玩家想玩，也让创作者看到「我也能造」。不是二选一。

## Product Purpose

InkWild 是一个 AI 驱动的互动叙事引擎。一套世界引擎，剧本 / 自由两种模式共享。已上线：双模式游玩、多 Agent 编排、创作工坊、多 LLM Provider 后台。

当前阶段目标：**网页版整体打磨，准备正式上线**。营销首页是第一门面——它要在几秒内让陌生人理解「这是什么、为什么值得玩」，并把人带进 discover / play。成功 = 陌生人落地首页后愿意开一局。

## Brand Personality

现代、大气、简约、电影感。

- **电影感** — 暗场 + 香槟金点睛 + 衬线大标题，像电影海报 / 片头，不像工具站。
- **大气** — 敢留白、敢用大标题、敢让单张封面占满屏；不堆信息、不塞指标。
- **简约** — 克制。金色是稀缺的点睛色（只用在品牌标识、精选、选中态、焦点环），不是到处刷的装饰。
- **现代** — 干净的暗色 UI，移动优先，动效服务叙事而非炫技。

声音：像一个懂故事的人在低声邀请你入场，不是 SaaS 在喊「立即注册」。

## Anti-references

- **通用 SaaS 落地页模板** — 蓝色渐变、hero 大指标数字（「10000+ 世界 / 99.9% 在线」这种）、千篇一律的三栏 feature 卡片网格、icon + 标题 + 一句话重复到底。InkWild 卖的是「故事」和「氛围」，不是「功能清单」。
- 顺带避开：电竞 RGB 霓虹风、卡通低龄糖果色、廉价 AI 味（紫粉渐变文字 / 玻璃拟态滥用 / 米色 AI 默认底）。

## Design Principles

1. **氛围先于信息** — 首页第一屏先给情绪（一张能把人吸进去的封面 + 一句有重量的话），再谈「它能做什么」。
2. **让位给故事** — UI 是舞台不是主角；chrome 越轻越好，金色越省越贵。
3. **给真东西，不给假指标** — 用真实世界 / 封面 / 剧本说话，禁止编造数据撑场面（「千篇一律的指标数字」是 SaaS 味的根）。
4. **移动端是原稿** — 先 375px 设计，桌面是放大版；触摸优先，不靠 hover 表达状态。
5. **两类用户一条动线** — 玩家「想玩」和创作者「想造」要在同一个门面里都成立。

## Accessibility & Inclusion

- **尊重 `prefers-reduced-motion`** — 大量 Framer Motion / scroll-snap / Ken Burns 动效必须有减弱降级；关键内容默认可见，不靠动画才出现。
- **移动优先 + 安全区** — `100dvh`、`safe-area-inset`、触摸目标 ≥ 44px。
- 基线对比度按 WCAG AA（暗底上的金 / 象牙文本，尤其正文 ≥ 4.5:1），作为默认而非额外要求。
